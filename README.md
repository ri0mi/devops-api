# devops-api

[![CI/CD](https://github.com/ri0mi/devops-api/actions/workflows/ci.yml/badge.svg)](https://github.com/ri0mi/devops-api/actions/workflows/ci.yml)

A minimal REST API built with FastAPI, used as a hands-on project to build a complete CI/CD pipeline: automated testing, containerization, image publishing, and deployment to a cloud server — all triggered by a single `git push`.

The API itself is intentionally small. The point of this project is the **pipeline around it**.

---

## What this project demonstrates

- Building and testing a Python REST API (FastAPI + pytest)
- Containerizing an application with a production-oriented Dockerfile
- Automating the full software delivery lifecycle with GitHub Actions
- Publishing images to Docker Hub
- Deploying automatically to an AWS EC2 instance over SSH
- Handling credentials securely with GitHub Actions secrets

---

## The pipeline

Every push to `main` triggers a three-stage pipeline. Each stage only runs if the previous one succeeds.

```
push to main
     │
     ▼
┌──────────┐     ┌─────────────────┐     ┌──────────┐
│   test   │ ──▶ │  build-and-push │ ──▶ │  deploy  │
└──────────┘     └─────────────────┘     └──────────┘
 run pytest       build Docker image      pull image on EC2
                  push to Docker Hub       restart container
```

1. **test** — spins up a clean runner, installs dependencies, and runs the test suite with pytest.
2. **build-and-push** — builds the Docker image and pushes it to Docker Hub. Runs only if tests pass.
3. **deploy** — connects to an EC2 instance over SSH, pulls the new image, and restarts the container. Runs only if the image was published.

The pipeline is defined in [`.github/workflows/ci.yml`](.github/workflows/ci.yml).

## API endpoints

| Method | Endpoint        | Description                          |
| ------ | --------------- | ------------------------------------ |
| GET    | `/`             | Root — returns a status message      |
| GET    | `/health`       | Health check (used by load balancers)|
| GET    | `/sum/{a}/{b}`  | Returns the sum of two integers      |

Interactive API docs are auto-generated by FastAPI at `/docs`.

## Tech stack

- **API:** FastAPI, Uvicorn
- **Testing:** pytest
- **Container:** Docker
- **CI/CD:** GitHub Actions
- **Registry:** Docker Hub
- **Hosting:** AWS EC2 (Amazon Linux 2023)

## Running locally

Clone the repo and install dependencies:

```bash
git clone https://github.com/ri0mi/devops-api.git
cd devops-api
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

Run the API:

```bash
uvicorn app.main:app --reload
```

The API is now available at `http://localhost:8000` (docs at `http://localhost:8000/docs`).

Run the tests:

```bash
pytest -v
```

---

## Running with Docker

Build the image:

```bash
docker build -t devops-api .
```

Run the container:

```bash
docker run -d -p 8000:8000 devops-api
```

The API is available at `http://localhost:8000`.

## Pipeline configuration

The deploy stage relies on the following GitHub Actions secrets:

| Secret               | Purpose                              |
| -------------------- | ------------------------------------ |
| `DOCKERHUB_USERNAME` | Docker Hub account for image push    |
| `DOCKERHUB_TOKEN`    | Docker Hub access token              |
| `EC2_HOST`           | Public address of the target server  |
| `EC2_USER`           | SSH user on the server               |
| `EC2_SSH_KEY`        | Private SSH key for deployment        |

## Notes and possible improvements

This project is a learning exercise, and a few things are simplified on purpose. Natural next steps toward a production setup:

- Use an **Elastic IP** so the server address stays fixed across restarts.
- Deploy via **AWS SSM Session Manager** instead of opening SSH to the internet.
- Add **linting** (e.g. ruff) and a **matrix** to test across multiple Python versions.
- Tag images by commit SHA instead of only `latest`, enabling rollbacks.
